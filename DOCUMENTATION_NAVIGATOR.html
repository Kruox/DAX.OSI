Welcome to DAX.OSI Plugin Development üëã

DAX.OSI is a cross-platform desktop application that simulates a custom operating system. As an external developer, you can create powerful plugins using the DOSI.Core library without needing access to the master source code.

What is DOSI.Core?

DOSI.Core is a reusable library published as a NuGet package containing:

  • Custom UI controls and window system
  • User management and authentication
  • Theme/accent management system
  • Notification system
  • Window management API
  • Terminal control components
  • Plugin interface definitions

Key Benefits

üé® Beautiful UI

Pre-built controls with consistent styling and theming

üîí Secure

User authentication and data isolation per user

üöÄ Easy to Use

Simple API for common tasks

üíª Cross-Platform

Works on Windows, Linux, and macOS

‚ö° Quick Start (5 Minutes)

Get a working plugin created and running in just 5 minutes:

Step 1: Create Your Project

mkdir ~/MyDAXPlugins
cd ~/MyDAXPlugins
dotnet new classlib -n MyFirstPlugin -f net9.0
cd MyFirstPlugin

Step 2: Add NuGet Packages

dotnet add package DOSI.Core
dotnet add package Avalonia --version 11.3.9

Step 3: Create Plugin Class

Replace the contents of Class1.cs with MyFirstPlugin.cs:

using DOSI.Core.Plugins;

namespace MyFirstPlugin;

public class MyFirstPlugin : IDosiPlugin
{
    public string PluginName => "My First Plugin";
    public string Version => "1.0.0";
    public string Author => "Your Name";
    public string Description => "My first DAX.OSI plugin";

    public Type GetMainWindowType() => typeof(MyFirstPluginWindow);
    public void OnLoad() => Console.WriteLine($"[{PluginName}] Loaded!");
    public void OnUnload() => Console.WriteLine($"[{PluginName}] Unloading...");
}

Step 4: Create Window Class

Create MyFirstPluginWindow.cs:

using DOSI.Core.Controls;
using Avalonia.Controls;
using Avalonia.Layout;

namespace MyFirstPlugin;

public class MyFirstPluginWindow : DosiPluginWindowUI
{
    public MyFirstPluginWindow()
    {
        WindowTitle = "My First Plugin";
        Width = 400;
        Height = 300;

        if (ContentArea != null)
        {
            var text = new TextBlock 
            { 
                Text = "Hello from my plugin!",
                FontSize = 24
            };
            ContentArea.Children.Add(text);
        }
    }
}

Step 5: Build and Deploy

dotnet build --configuration Release
# Copy to: ~/path/to/DOSI/Users/YourUsername/Applications/MyFirstPlugin.dll

Step 6: Test

  1. Open DAX.OSI application
  2. Log in with any user
  3. Open Terminal (Ctrl+T)
  4. Type plugins to see your plugin listed
  5. Click Applications menu to launch your plugin
‚ú® Congratulations! You've created your first DAX.OSI plugin! Now explore the detailed sections to add more features.

üì¶ Plugin Structure

A valid DAX.OSI plugin must consist of exactly two classes:

1. Plugin Main Class

Implements the IDosiPlugin interface:

public interface IDosiPlugin
{
    string PluginName { get; }      // Display name shown to users
    string Version { get; }         // Semantic version (e.g., "1.0.0")
    string Author { get; }          // Your name
    string Description { get; }     // What the plugin does
    
    Type GetMainWindowType();       // Return your window class type
    void OnLoad();                  // Called when plugin loads
    void OnUnload();                // Called when plugin unloads
}

2. Plugin Window Class

Inherits from DosiPluginWindowUI and implements the UI:

public class MyPluginWindow : DosiPluginWindowUI
{
    // ContentArea property is where you add controls
    // WindowTitle property sets the window title
    // Width and Height set window dimensions
    
    public MyPluginWindow()
    {
        WindowTitle = "My Plugin";
        Width = 500;
        Height = 400;
        
        // Add UI controls to ContentArea
        if (ContentArea != null)
        {
            var button = new Button { Content = "Click Me" };
            ContentArea.Children.Add(button);
        }
    }
}

üéØ Common Patterns

Pattern 1: Data Storage

public class MyPlugin : IDosiPlugin
{
    private string _dataFolder = Path.Combine(
        SystemSettings.SystemFolder,
        "Users",
        UserManager.Instance.CurrentUser?.Username ?? "User",
        "Applications",
        "MyPluginData"
    );

    public void OnLoad()
    {
        Directory.CreateDirectory(_dataFolder);
    }

    public void SaveData(string filename, string content)
    {
        var path = Path.Combine(_dataFolder, filename);
        File.WriteAllText(path, content);
    }
}

Pattern 2: Theme-Aware UI

using DOSI.Core.Accent;

public class MyWindow : DosiPluginWindowUI
{
    public MyWindow()
    {
        var theme = AccentManager.Instance.CurrentTheme;
        
        var text = new TextBlock
        {
            Foreground = new SolidColorBrush(theme.TextColor),
            Background = new SolidColorBrush(theme.PrimaryColor)
        };
        
        // Listen for theme changes
        AccentManager.Instance.ThemeChanged += (s, newTheme) =>
        {
            text.Foreground = new SolidColorBrush(newTheme.TextColor);
        };
    }
}

Pattern 3: Notifications

using DOSI.Core.Notifications;

private async void ShowNotification()
{
    await NotificationHelper.Notify("Action completed!");
}

Pattern 4: User-Specific Operations

var currentUser = UserManager.Instance.CurrentUser;
if (currentUser != null)
{
    var username = currentUser.Username;
    var accent = currentUser.Accent;
}

üíª Code Examples

Example 1: Simple Counter Plugin

using DOSI.Core.Plugins;
using DOSI.Core.Controls;
using Avalonia.Controls;
using Avalonia.Layout;

namespace CounterPlugin;

public class CounterPlugin : IDosiPlugin
{
    public string PluginName => "Counter";
    public string Version => "1.0.0";
    public string Author => "Developer";
    public string Description => "A simple counter application";

    public Type GetMainWindowType() => typeof(CounterWindow);
    public void OnLoad() => Console.WriteLine("[Counter] Loaded!");
    public void OnUnload() => Console.WriteLine("[Counter] Unloading...");
}

public class CounterWindow : DosiPluginWindowUI
{
    private int _count = 0;
    private TextBlock? _display;

    public CounterWindow()
    {
        WindowTitle = "Counter";
        Width = 300;
        Height = 200;
        InitializeUI();
    }

    private void InitializeUI()
    {
        if (ContentArea == null) return;

        var stack = new StackPanel { Spacing = 20, Padding = new Avalonia.Thickness(20) };

        _display = new TextBlock
        {
            Text = "0",
            FontSize = 48,
            TextAlignment = Avalonia.Media.TextAlignment.Center
        };
        stack.Children.Add(_display);

        var button = new Button { Content = "Increment" };
        button.Click += (s, e) =>
        {
            _count++;
            _display.Text = _count.ToString();
        };
        stack.Children.Add(button);

        ContentArea.Children.Add(stack);
    }
}

üêõ Debugging Your Plugin

Enable Console Logging

public class MyPlugin : IDosiPlugin
{
    private const string LogPrefix = "[MyPlugin]";

    private void Log(string message)
    {
        Console.WriteLine($"{LogPrefix} {message}");
    }

    public void OnLoad()
    {
        Log("Plugin initialized");
        Log($"System folder: {SystemSettings.SystemFolder}");
        Log($"Current user: {UserManager.Instance.CurrentUser?.Username}");

‚ùå Troubleshooting

Problem: Plugin not loading

Symptom: "plugins" command doesn't show your plugin
Solution: Verify DLL is in correct folder, check console for errors

Problem: "Could not load file or assembly"

Cause: Version mismatch or missing dependency
Solution: Ensure project targets .NET 9.0, update NuGet packages

Problem: "Type does not implement IDosiPlugin"

Cause: Interface not properly implemented
Solution: Check all properties are implemented, verify namespace

Problem: Window doesn't appear

Cause: ContentArea is null or no controls added
Solution: Check ContentArea != null before adding controls

Problem: Crashes on plugin launch

Cause: Exception in plugin code
Solution: Check console output for exception details, add try-catch

‚úÖ What You CAN Do

As an external plugin developer, you have full access to:

  • ‚úÖ Create custom window-based applications
  • ‚úÖ Use all DOSI.Core UI controls
  • ‚úÖ Access user profile information (read-only)
  • ‚úÖ Use accent/theme system
  • ‚úÖ Create and manage data files in user directories
  • ‚úÖ Use notification system
  • ‚úÖ Access window management API
  • ‚úÖ Build rich UIs with Avalonia controls
  • ‚úÖ Use all .NET 9.0 capabilities
  • ‚úÖ Distribute plugins independently
  • ‚úÖ Monetize your plugins
  • ‚úÖ Open-source your plugins

Example: Full API Access

// Access user information
var user = UserManager.Instance.CurrentUser;
var username = user?.Username;

// Access accent system
var theme = AccentManager.Instance.CurrentTheme;
var primaryColor = theme.PrimaryColor;

// Access system settings
var systemFolder = SystemSettings.SystemFolder;

// Show notifications
await NotificationHelper.Notify("Hello!");

// Create user-specific files
var docsFolder = Path.Combine(systemFolder, "Users", username, "Documents");
File.WriteAllText(Path.Combine(docsFolder, "file.txt"), "content");

‚ùå What You CANNOT Do

The following operations are restricted to protect system integrity:

  • ‚ùå Create built-in applications (requires master source)
  • ‚ùå Add system terminal commands
  • ‚ùå Modify Boot, Login, or Desktop screens
  • ‚ùå Create new accent themes
  • ‚ùå Modify user management system
  • ‚ùå Modify file system structure
  • ‚ùå Modify window manager functionality
  • ‚ùå Access private members of DOSI.Core classes
  • ‚ùå Hook into internal event systems

Why These Limits?

  • üîí Ensures system stability and security
  • üõ°Ô∏è Prevents plugins from breaking core functionality
  • üë• Protects user data integrity
  • üé® Maintains consistent UI/UX experience
  • üîÑ Allows DAX.OSI to be updated safely
üí° Note: If you need features beyond these limits, the master source is available for core contributors.

üîß Creating Installers

Windows Batch Installer

@echo off
REM MyFirstPlugin Installer

setlocal enabledelayedexpansion

REM Find DOSI folder
if exist "%APPDATA%\DOSI" (
    set DOSI_PATH=%APPDATA%\DOSI
) else if exist "DOSI" (
    set DOSI_PATH=DOSI
) else (
    echo Error: Could not find DOSI folder
    pause
    exit /b 1
)

set USERNAME=%USERNAME%
set INSTALL_PATH=!DOSI_PATH!\Users\!USERNAME!\Applications

if not exist "!INSTALL_PATH!" mkdir "!INSTALL_PATH!"

copy "MyFirstPlugin.dll" "!INSTALL_PATH!\"

echo.
echo Installation Complete!
echo Location: !INSTALL_PATH!\MyFirstPlugin.dll
echo.
echo Next steps:
echo 1. Open DAX.OSI
echo 2. Log in
echo 3. Open Terminal (Ctrl+T)
echo 4. Type 'plugins' to verify
echo.
pause

Linux/Mac Shell Installer

#!/bin/bash
# MyFirstPlugin Installer

# Find DOSI folder
if [ -d "$HOME/.local/share/DOSI" ]; then
    DOSI_PATH="$HOME/.local/share/DOSI"
elif [ -d "DOSI" ]; then
    DOSI_PATH="DOSI"
else
    echo "Error: Could not find DOSI folder"
    exit 1
fi

USERNAME=$(whoami)
INSTALL_PATH="$DOSI_PATH/Users/$USERNAME/Applications"

mkdir -p "$INSTALL_PATH"
cp MyFirstPlugin.dll "$INSTALL_PATH/"

echo "Installation Complete!"
echo "Location: $INSTALL_PATH/MyFirstPlugin.dll"
echo ""
echo "Next steps:"
echo "1. Open DAX.OSI"
echo "2. Log in"
echo "3. Open Terminal (Ctrl+T)"
echo "4. Type 'plugins' to verify"
